Initial steps

Research, research, research! The first part of this project was to understand what was needed and why. I interviewed stakeholders, researched different styles and trends are started out with some initial direction illustrations. These were meant to be a good jumping off point for the stakeholders to choose a direction.

Extending the style

The next step I took was to extend the style we moved forward with into different uses. I experimented with how to illustrate different topics and used different fidelities of illustration. The style we moved forward with portrayed the feeling of using the product more than the actual product itself, so I used happy metaphors to represent how using it would make you feel. All the illustrations are human-first, friendly, and diverse: the things that the stakeholders wanted to express. After that, I did some sketches to see how to extend hero illustrations to spot illustrations and icons.

Experimenting with color

Once the style and system was in a good place, it was time to move to color. This was a challenging one because it needed to fit in with the site and style, but also feel exciting and fresh. What I settled on was a color system that uses GitHub's Primer color library. Primer is the library of colors to be used for all GitHub products. It's very primary, but after searching through the database to find colors that complemented each other, I found several colors that worked really well together. I created a modular system of colors that can be exchanged for others while continuing to feel consistent. This way the illustrations have more colors to change the style and feeling, but each one still matches both each other and the site.

Integrating illustrations into the web design

When illustrating for web, you should keep web in mind throughout the whole process. I did each sketch over a screenshot of the web page to make sure it fit in well. After that it was a question of how to integrate even more. An issue that popped up was that the illustrations didn't match the blue icons we used occasionally. To fix that I created a set of blue decorations that matched the color and line width of the icons and put them around each illustration. This pushed the integration further and had them match not only the Primer colors, but also the icons.

Site Illustrations

This project tasked me with the large challenge of creating a completely new system of illustrations to complement a potential site rebrand.

The purpose of the project was to create a series of friendly, inclusive, exciting illustrations to introduce people to GitHub. To do this I created a style that was bright, diverse, and metaphorical. I wanted to not only to representational, but aspirational when planning these illustrations, so I made sure to include a majority women and POC in most illustrations and give women more leading roles.
